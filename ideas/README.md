# ideas

The front door. Dump ideas here, one markdown file per idea.

## How to dump

- One file per idea. Name it a short kebab-case slug (`topic-feeds.md`, `recipe-box.md`).
- No required shape. A title and a sentence is a complete idea. Grow it later if it earns the attention.
- Do not overthink it. A half-formed idea written down beats a good idea lost.
- Cross-link related ideas with plain markdown links.

## Lifecycle

An idea here is a seed, not a commitment. Three things can happen to it:

- **Graduates** - it becomes real and moves into `apps/` (or its own repo once stable).
- **Merges** - it folds into another idea.
- **Drops** - it stops being interesting and gets deleted. That is fine and expected.

Git history keeps the dropped ones if they ever matter again.

## Suggested file shape (optional)

```markdown
# <idea name>

**What** - one or two sentences.

**Why it fits the Atmosphere** - which atproto primitive it leans on (feed generator, custom lexicon, labeler, repo-as-storage).

**Open questions** - whatever is still fuzzy.
```
